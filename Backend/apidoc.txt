# Property Listing API Documentation

**Local dev base URL (default `python manage.py runserver`):** `http://127.0.0.1:8844` — this project avoids **8000** because another Django app often uses it and causes **404** on `/api/accounts/...`. Override: `python manage.py runserver 127.0.0.1:8000` or set env `P4U_DEV_PORT`.

## Authentication APIs

**Sanity check (before calling register):** `GET {base}/api/_meta/` (e.g. `GET http://127.0.0.1:8844/api/_meta/`) must return JSON with `"ROOT_URLCONF": "property_listing.urls"` and `"register_path": "/api/accounts/register/"`. If you get 404 or a different `ROOT_URLCONF`, you are not hitting this backend (wrong host/port or another process).

Canonical paths from **`apidoc.txt`** are wired in **`property_listing/urls.py`**. Optional aliases live in **`accounts/urls.py`** (included at `api/accounts/`).

### Accounts API index
- **URL**: `/api/accounts/`
- **Method**: GET
- **Auth Required**: No
- **Success Response**: **200 OK** — JSON with `detail`, `ROOT_URLCONF`, and an `endpoints` map (register, login, profile, OTP routes, etc.). Use to confirm routing in dev.

### `UserSerializer` (auth responses)

Returned as `user` on login/register/verify and as the body of `GET`/`PATCH` profile. Owner accounts include profile fields from `UserProfile`.

| Field | Type | Notes |
|-------|------|--------|
| `id` | integer | Django user pk |
| `username` | string | |
| `email` | string | |
| `first_name`, `last_name` | string | Display name; profile PATCH accepts `full_name` to update both |
| `is_staff` | boolean | Admin/staff |
| `is_property_owner` | boolean | Has `UserProfile` with `account_type` = `property_owner` |
| `phone` | string | From profile |
| `whatsapp_number` | string | From profile |
| `address` | string | From profile (owner dashboard) |
| `avatar` | string \| null | Absolute URL when set |
| `account_type` | string \| null | e.g. `property_owner` |
| `email_verified` | boolean | `true` when `profile.email_verified_at` is set |

**Example `user` object** (owner after verify, OTP login, or profile GET):
```json
{
  "id": 12,
  "username": "owner_jane",
  "email": "owner@example.com",
  "first_name": "Jane",
  "last_name": "Owner",
  "is_staff": false,
  "is_property_owner": true,
  "phone": "+971501234567",
  "whatsapp_number": "+971509876543",
  "address": "123 Marina Walk, Dubai",
  "avatar": "http://127.0.0.1:8844/media/avatars/photo.webp",
  "account_type": "property_owner",
  "email_verified": true
}
```

### Register Admin Account
- **URL**: `/api/accounts/register/`
- **Method**: POST
- **Auth Required**: No
- **Request Body**:
  ```json
  {
    "username": "admin",
    "email": "admin@example.com",
    "password": "StrongPassword123!",
    "password2": "StrongPassword123!",
    "first_name": "Admin",
    "last_name": "User"
  }
  ```
- **Success Response**: 
  - **Code**: 201 CREATED
  - **Content**: 
    ```json
    {
      "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b",
      "user": {
        "id": 1,
        "username": "admin",
        "email": "admin@example.com",
        "first_name": "Admin",
        "last_name": "User",
        "is_staff": true,
        "is_property_owner": false,
        "phone": "",
        "whatsapp_number": "",
        "address": "",
        "avatar": null,
        "account_type": null,
        "email_verified": false
      }
    }
    ```
- **Error Response**: 
  - **Code**: 400 BAD REQUEST
  - **Content**: Validation errors
- **Aliases (optional, same request body and `RegisterView`)**: `POST /api/accounts/register/user/`, `POST /api/accounts/register/admin/`
- **404 troubleshooting**: The Django 404 page should reference **`property_listing.urls`**. If you see another project name, stop that server and run `python manage.py runserver` from this repo’s **`Backend`** folder. Use **JSON** body and `Content-Type: application/json` in clients (e.g. Postman).

### Update owner profile
- **URL**: `/api/accounts/profile/`
- **Method**: PATCH
- **Auth Required**: Yes (Token) — property owner with `UserProfile` only
- **Headers**: `Authorization: Token <token>`, `Content-Type: application/json`

| Field | Required | Type | Notes |
|-------|----------|------|--------|
| `full_name` | No | string | Updates `first_name` + `last_name` |
| `phone` | No | string | |
| `whatsapp_number` | No | string | |
| `address` | No | string | |
| `current_password` | Conditional | string | Required with password change (send all three password fields together) |
| `new_password` | Conditional | string | Django validators |
| `new_password2` | Conditional | string | Must match `new_password` |
| `email` | Conditional | string | Requires `email_otp` after `POST .../profile/email-change/request/` |
| `email_otp` | Conditional | string | 6-digit OTP sent to **new** email |

- **Request Body** (all fields optional unless noted; send only what you change):
  ```json
  {
    "full_name": "Jane Owner",
    "phone": "+971501234567",
    "whatsapp_number": "+971509876543",
    "address": "123 Marina Walk, Dubai, UAE"
  }
  ```
- **Change password** (send all three together):
  ```json
  {
    "current_password": "OldPass123!",
    "new_password": "NewPass456!",
    "new_password2": "NewPass456!"
  }
  ```
- **Change email** (after OTP request below; send `email` and `email_otp` together):
  ```json
  {
    "email": "newemail@example.com",
    "email_otp": "123456"
  }
  ```
- **Success Response**:
  - **Code**: 200 OK
  - **Content**: Updated `UserSerializer` (same shape as GET profile)
- **Error Response**:
  - **Code**: 400 BAD REQUEST — field validation (e.g. wrong `current_password`, missing `email_otp` when email changed)
  - **Code**: 403 FORBIDDEN — no owner profile

### Request email-change OTP (profile)
- **URL**: `/api/accounts/profile/email-change/request/`
- **Method**: POST
- **Auth Required**: Yes (Token) — property owner with `UserProfile` only
- **Headers**: `Authorization: Token <token>`, `Content-Type: application/json`
- **Request Body**:
  ```json
  {
    "new_email": "newemail@example.com"
  }
  ```
- **Success Response**:
  - **Code**: 200 OK
  - **Content**: `{"success": true, "message": "OTP sent successfully.", "otp": "123456"}`
- **Notes**: Emails a 6-digit OTP to the **new** address only; on success, stores `pending_email` on the profile. Response includes `otp` for client/testing. Complete the change with **PATCH** `/api/accounts/profile/` including `email` and `email_otp`. OTP expires in 10 minutes. OTP is never sent to admin email.
- **Error Response**:
  - **Code**: 400 BAD REQUEST — field validation, e.g. `{"new_email": ["Email already in use."]}`
  - **Code**: 400 BAD REQUEST — delivery / send failure: `{"success": false, "message": "Unable to send OTP to this email address."}`

### Owner register — step 1 (send OTP)
- **URL**: `/api/accounts/register/owner/init/`
- **Method**: POST
- **Auth Required**: No
- **Content-Type**: `multipart/form-data`

| Field | Required | Type | Notes |
|-------|----------|------|--------|
| `full_name` | **Yes** | string | Split into `first_name` / `last_name` on the user record |
| `email` | **Yes** | string | Must be unique (case-insensitive) |
| `phone` | **Yes** | string | Stored on `UserProfile.phone` |
| `whatsapp_number` | **Yes** | string | Stored on `UserProfile.whatsapp_number` |
| `password` | **Yes** | string | Django password validators apply |
| `password2` | **Yes** | string | Must match `password` |
| `avatar` | No | file | Alias: `profile_photo`. Max **500 KB**; jpg/jpeg/png/webp |

- **Success Response** — **201 CREATED**:
  ```json
  { "success": true, "message": "OTP sent successfully.", "otp": "123456" }
  ```
  No `token` until step 2. Creates **inactive** `User` + `UserProfile`; emails 6-digit OTP to the entered email only (`purpose`: `registration`). Rolls back user creation if email delivery fails. Response includes `otp`.
- **Error Response** — **400 BAD REQUEST** (examples):
  ```json
  { "email": ["Email already in use."] }
  ```
  ```json
  { "password": ["Password fields didn't match."] }
  ```
  ```json
  { "success": false, "message": "Unable to send OTP to this email address." }
  ```
  Field-level validation errors for missing/invalid fields.

### OTP purposes (`OTPVerification.purpose`)
| Value | Used for |
|-------|----------|
| `registration` | Owner sign-up (`register/owner/init` → `verify`) |
| `login` | Owner OTP login |
| `password_reset` | Forgot password flow |
| `email_change` | Profile email change (`profile/email-change/request` → `PATCH profile` with `email_otp`) |

OTP codes expire after **10 minutes**.

#### List All Properties
- **URL**: `/api/properties/properties/`
- **Method**: GET
- **Auth Required**: No (public). Optional `Authorization: Token <token>` for staff-only filters.
- **Pagination**: Page-number (`page`, `page_size`, default `page_size=10`, max `100`)
- **Public visibility**: Unauthenticated and non-staff users only receive properties with `moderation_status` = `approved` (pending and rejected listings are hidden).
- **Query Parameters**:
  - `moderation_status` &mdash; **Staff only** (`is_staff`): `pending`, `approved`, `rejected`, or `all` (default). Filters the admin list; ignored for non-staff (they always get approved-only).
  - `property_for` &mdash; `rent` or `sell`
  - `price_min`, `price_max` &mdash; Decimal amounts
  - `ownership` &mdash; `management` or `direct_owner`
  - `area_min`, `area_max` &mdash; Numeric; combine with `area_unit` (`sqft` default, `cent` converted to sqft)
  - `date_from`, `date_to` &mdash; `YYYY-MM-DD` (applies to `created_at`)
  - `property_type` &mdash; Property type ID
  - `bedrooms_min`, `bedrooms_max`
  - `bathrooms_min`, `bathrooms_max`
  - `state_id`, `district_id`, `city_id`
  - `location` &mdash; Text match against state/district/city names
  - `latitude`, `longitude` &mdash; Nearby filter: properties within `radius` km (defaults to admin `filter_radius` from site settings when `radius` omitted)
  - `radius` &mdash; Optional override for nearby filter radius in kilometers (used with `latitude` + `longitude`)
  - `lat_min`, `lat_max`, `lng_min`, `lng_max` &mdash; Bounding-box coordinate filters
  - `furnishing` &mdash; Exact furnishing value (case-insensitive)
  - `search` &mdash; Matches property title, description, feature names, or nearby places
  - `include_ads` &mdash; Optional boolean (default `false`). When `true`, response `results` is a **mixed feed** of `{ "type": "property", "data": {...} }` and `{ "type": "ad", "data": {...} }` items. | Field | Type | Notes |

| `youtube_video_link` | string | null |
| `property_video` | file | Write-only on create/update (multipart). mp4, mov, avi, mkv, webm. After save, the server runs ffmpeg to extract a frame and stores it as `video_thumbnail` on the model; responses expose `property_video_url` and `video_thumbnail_url` only. If ffmpeg is unavailable, the upload still succeeds but `video_thumbnail_url` stays `null`. |
| `nearby_places_data` | JSON string or array | `[{ "name": "School", "distance": 0.5 }]` — `distance` in km |
| `is_featured` | boolean | `false` |
| `uploaded_images` | file(s) | Repeat field for multiple images |

- **Example multipart body** (conceptual):
  ```
  property_for=rent
  property_ownership=direct_owner
  state=1
  # contact_name, whatsapp_number, phone_number, email optional for owners — filled from profile when omitted
  district=2
  city=3
  title=Luxury Apartment in Downtown
  price=1500.00
  property_type=1
  bedrooms=2
  bathrooms=2
  area=1200
  area_unit=sqft
  description=Beautiful apartment with city view
  built_year=2018
  furnishing=furnished
  parking_spaces=1
  features=1
  features=2
  google_maps_url=https://maps.google.com/?q=Kochi
  nearby_places_data=[{"name":"International School","distance":0.5}]
  uploaded_images=<file1>
  uploaded_images=<file2>
  ```
- **Example cent listing** (built fields omitted):
  ```
  property_for=sell
  property_ownership=direct_owner
  state=1
  district=2
  city=3
  title=5 Cent Plot Near Highway
  price=45 Lakh
  property_type=3
  area=5
  area_unit=cent
  description=Clear title, road access
  ```
- **Success Response** — **201 CREATED**: Full property object (see **Property object (response shape)**). Owner example includes `"moderation_status": "pending"`, `"created_by": <user id>`. Cent listings may return `"bedrooms": null`, `"bathrooms": null`, `"built_year": null`, `"furnishing": null`.
- **Error Response** — **400 BAD REQUEST** (examples):
  ```json
  { "state": ["State is required"] }
  ```
  ```json
  { "uploaded_images": ["Could not compress image within size limit."] }
  ```

#### Update Property
- **URL**: `/api/properties/properties/{id_or_slug}/`
- **Method**: PATCH
- **Auth Required**: Yes (Token) — **staff** may edit any listing; **owners** may edit only `created_by` = self
- **Headers**: `Authorization: Token <token>`
- **Content-Type**: `multipart/form-data`
- **Request Body**: Any subset of create fields (all optional on PATCH). Replace nearby places by sending full `nearby_places_data` array. Append images with `uploaded_images`.
- **`area_unit` on PATCH**: Changing to `cent` clears `bedrooms`, `bathrooms`, `built_year`, and `furnishing` to `null` when those fields are omitted in the same request. Changing to `sqft` requires `bedrooms`, `bathrooms`, and `furnishing` (in the request or already stored on the listing); `built_year` remains optional.
- **Success Response** — **200 OK**: Full updated property object (same shape as create response).
- **Error Response**: **404** if not found or not owned by caller; **400** validation errors.

## Advertisement Management APIs

Base path: `/api/advertisements/`

### Property feed with injected ads

When `include_ads=true` on `GET /api/properties/properties/`, each page of `results` may contain interleaved ads. Default is plain property objects (no ads).

**Mixed result item shapes**

```json
{
  "count": 27,
  "next": "http://127.0.0.1:8844/api/properties/properties/?page=2",
  "previous": null,
  "results": [
    { "type": "property", "data": { "id": 1, "title": "...", "...": "..." } },
    { "type": "property", "data": { "id": 2, "title": "...", "...": "..." } },
    { "type": "property", "data": { "id": 3, "title": "...", "...": "..." } },
    { "type": "property", "data": { "id": 4, "title": "...", "...": "..." } },
    { "type": "property", "data": { "id": 5, "title": "...", "...": "..." } },
    {
      "type": "ad",
      "data": {
        "id": 1,
        "title": "Summer Property Promo",
        "ad_type": "property",
        "media_type": "image",
        "desktop_image_url": "http://127.0.0.1:8844/media/ads/images/desktop/promo.webp",
        "redirect_type": "property",
        "linked_property": 42
      }
    },
    { "type": "property", "data": { "id": 6, "...": "..." } }
  ]
}
```

- `count`, `next`, and `previous` still refer to **properties only** (ads do not affect pagination totals).
- Ads are selected from active ads eligible for the property listing feed. Without location params: property ads first, then generic. With location params: generic ads always included; property ads filtered by radius/region; property ads ordered before generic.
- One ad is inserted per slot, rotating through eligible ads (property ads before generic, then priority and creation order). The gap between slots uses the global `ad_inject_after_every_n_properties` from site settings (default 5).
- Default / legacy clients: `GET /api/properties/properties/` without `include_ads=true` returns plain property objects in `results`.
